## 第四部分：缓存与 CDN 策略

### 缓存头部汇总

| API 端点 | 缓存层 | Vercel | Netlify | 一致 |
|---|---|---|---|---|
| `/api/v1/today.json` | 浏览器 | `max-age=3600` (1h) | `max-age=3600` (1h) | ✅ |
| `/api/v1/today.json` | CDN | **缺失** | `s-maxage=86400` (24h) | ❌ |
| `/api/v1/today.json` | SWR | **缺失** | `stale-while-revalidate=3600` (1h) | ❌ |
| `/api/v1/theme/{preset}.json` | 浏览器 | `max-age=86400` (1d) | `max-age=86400` (1d) | ✅ |
| `/api/v1/theme/{preset}.json` | CDN | **缺失** | `s-maxage=31536000` (1y) | ❌ |

### Vercel 缺失项

```
s-maxage=86400              ← CDN 边缘节点缓存 24 小时
stale-while-revalidate=3600  ← 缓存过期后 1 小时内后台静默刷新
s-maxage=31536000           ← 预设主题 CDN 永久缓存
```

### 影响评估

| 影响 | 严重度 | 说明 |
|---|---|---|
| 源站负载 | 低 | Vercel Serverless 自动伸缩，但每次请求触发函数执行 |
| 响应延迟 | 低 | 缺少边缘缓存，用户可能感受到毫秒级额外延迟 |
| 功能正确性 | 无 | 不影响数据准确性 |

### 建议

在 Vercel 项目的 `vercel.json` 中添加 headers 配置：

```json
{
  "headers": [
    {
      "source": "/api/v1/today.json",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=3600, s-maxage=86400, stale-while-revalidate=3600" }
      ]
    },
    {
      "source": "/api/v1/theme/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=86400, s-maxage=31536000, immutable" }
      ]
    }
  ]
}
```
