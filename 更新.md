# ThemeDist Monitor API 文档

> 更新日期：2026-05-25

本站点（themedist-monitor）是 ThemeDist 主题分发平台的实时监控台。以下列出本站点对外**提供**的全部 API 端点。

---

## 端点总览

| # | 端点 | 方法 | CORS | 说明 |
|---|------|------|------|------|
| 1 | `/api/v1/status` | GET, OPTIONS | 是 | 双平台实时状态 + 最新主题快照 |
| 2 | `/api/v1/data` | GET | 否 | 综合仪表盘数据（含 SLA、安全事件） |
| 3 | `/api/v1/monitor` | GET, POST, DELETE | 否 | 触发全量监控检查 |
| 4 | `/api/v1/probe` | GET | 否 | 多区域拨测（Vercel Cron 06:00 UTC） |
| 5 | `/api/v1/telemetry` | POST | 否 | RUM 用户端页面加载耗时上报 |
| 6 | `/api/v1/today-safe` | GET | 是 | 安全代理：经清理/扫描/校验的 today.json |
| 7 | `/api/v1/security-status` | GET | 是 | 主题安全状态摘要 |
| 8 | `/api/v1/alerts/resolve` | POST | 否 | 告警处理（单条或批量） |
| 9 | `/api/v1/badges/[type]` | GET | 是 | SVG 状态徽章（可嵌入 README） |
| 10 | `/api/v1/diagnose` | GET | 否 | 服务端网络连通性诊断（含限流） |
| 11 | `/api/v1/debug-kv` | GET | 否 | KV 存储读写测试（运维调试） |

## 1. 平台实时状态 — `/api/v1/status`

**方法：** `GET` `OPTIONS`
**缓存：** `s-maxage=30, stale-while-revalidate=120`
**CORS：** `Access-Control-Allow-Origin: *`

### 响应

```json
{
  "overall": "healthy",
  "platforms": {
    "vercel": {
      "status": "online",
      "latencyMs": 486,
      "cacheStatus": "MISS",
      "error": null
    },
    "netlify": {
      "status": "online",
      "latencyMs": 1778,
      "cacheStatus": "HIT",
      "error": null
    }
  },
  "theme": {
    "date": "2026-05-25",
    "presetName": "DONT PANIC",
    "themeCount": 151,
    "isSafe": true
  },
  "checkedAt": "2026-05-25T08:23:28.072Z",
  "_edgeHash": {
    "vercel:status": "online",
    "vercel:latency": 486,
    "vercel:cache": "MISS",
    "netlify:status": "online",
    "netlify:latency": 1778,
    "netlify:cache": "HIT",
    "db:status": "healthy",
    "checkedAt": "2026-05-25T08:23:28.072Z"
  }
}
```

### 字段说明

| 字段 | 类型 | 可能值 |
|------|------|--------|
| `overall` | string | `healthy` / `degraded` / `down` / `unknown` |
| `platforms.<p>.status` | string | `online` / `slow` (延迟 >2s) / `outage` / `unknown` |
| `platforms.<p>.cacheStatus` | string | `HIT` / `MISS` / `BYPASS` / `UNKNOWN` |
| `platforms.<p>.latencyMs` | number \| null | 最近一次请求延迟（毫秒） |
| `platforms.<p>.error` | string \| null | 错误信息，正常时为 null |
| `theme.isSafe` | boolean | 主题安全扫描是否通过 |
| `_edgeHash` | object | KV status hash 原值，前端未使用 |

### OPTIONS (CORS 预检)

返回 `204 No Content`，响应头包含：
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, OPTIONS
Access-Control-Allow-Headers: Content-Type
Access-Control-Max-Age: 86400
```

## 3. 触发监控检查 — `/api/v1/monitor`

### GET — 公开触发（含限流）

```
GET /api/v1/monitor
```

执行全量监控检查（性能探测 + 主题安全扫描 + 社区主题审计），限流频率由 `rate-limit.ts` 控制。

超限返回：
```json
{ "error": "Rate limited", "retryAfter": 249 }
```
HTTP 状态码 `429`，响应头含 `Retry-After`。

并发锁（`lock:monitor`，TTL 60s）防止重复执行，锁冲突时返回：
```json
{ "error": "Monitor is already running", "retryAfter": 60 }
```
HTTP 状态码 `429`。

### POST — Cron 触发

```
POST /api/v1/monitor
Authorization: Bearer <CRON_SECRET>
```

行为与 GET 相同，需 Bearer Token 鉴权。由 Vercel Cron 每日 00:00 UTC 自动调用。

无有效 Token 返回：
```json
{ "error": "Unauthorized" }
```
HTTP 状态码 `401`。

### DELETE — 清空监控数据

```
DELETE /api/v1/monitor
Authorization: Bearer <CRON_SECRET>
```

删除 `/tmp/data`（Vercel）或 `data/`（本地）目录下全部监控数据文件。

响应：
```json
{ "message": "All monitoring data cleared" }
```

**运行时：** Edge（`export const runtime = 'edge'`）

### 响应

```json
{
  "probe": { "region": "SG", "duration": 10449 },
  "results": [
    {
      "id": "8901d06d-ff6d-4fd2-be2d-fa5d437c9ed8",
      "timestamp": "2026-05-25T08:24:36.884Z",
      "region": "SG",
      "platform": "vercel",
      "endpoint": "https://themedist.vercel.app/api/v1/today.json",
      "statusCode": 0,
      "latencyMs": 10009,
      "error": "This operation was aborted"
    },
    {
      "id": "21a77c77-1e3f-41c4-814b-59b03255f359",
      "timestamp": "2026-05-25T08:24:27.326Z",
      "region": "SG",
      "platform": "netlify",
      "endpoint": "https://themedist.netlify.app/api/v1/today.json",
      "statusCode": 200,
      "latencyMs": 448
    }

```

### 字段说明

| 字段 | 说明 |
|------|------|
| `probe.region` | 边缘节点地区（取自 `x-vercel-ip-country` 请求头，本地为 `"unknown"`） |

## 5. RUM 遥测上报 — `/api/v1/telemetry`

**方法：** `POST`

接收用户浏览器端页面加载耗时数据。按「平台 + 日期 + 延迟分桶」聚合写入 KV（`rum:counters` / `rum:latency_bins`），**不存储原始条目**。写入为 fire-and-forget。

### 请求体

```json
{
  "duration": 320,
  "platform": "vercel"
}
```

| 字段 | 类型 | 约束 |
|------|------|------|
| `duration` | number | 必填，0–60000（毫秒） |
| `platform` | string | 可选，默认 `"unknown"` |

### 响应

**成功：** `200` `{ "ok": true }`

**无效 duration：** `400` `{ "error": "Invalid duration" }`

**请求体解析失败：** `400` `{ "error": "Bad request" }`

### 延迟分桶

| 桶 | 范围 |
|----|------|
| `0-100` | ≤100ms |
| `101-250` | 101–250ms |
| `251-500` | 251–500ms |
| `501-1000` | 501–1000ms |
| `1001-2000` | 1001–2000ms |
| `2001+` | >2000ms |

### KV 写入项

每条上报写入 4 个哈希字段（均为原子递增）：
- `{platform}:{date}:total` +1
- `{platform}:{date}:ok` +1（duration < 2000 时）
- `{platform}:{date}:sum_latency` +duration
- `{platform}:{date}:{latencyBin}` +1

KV 未配置时直接返回 `{ "ok": true }`，不做写入。

## 6. 安全代理 — `/api/v1/today-safe`

**方法：** `GET`
**CORS：** `Access-Control-Allow-Origin: *`

### 处理流水线

```
获取 (Vercel → Netlify 依次尝试)
  → 清理 (HTML 剥离)
  → 扫描 (XSS / prototype pollution / 注入特征)
  → 校验 (Schema 结构)
  → 通过 → 缓存至 KV (theme:last_safe) → 返回
  → 失败 → 回退至 KV 中的上次安全快照
```

### 成功响应

```json
{
  "date": "2026-05-25",
  "generatedAt": "2026-05-25T07:29:19.537Z",
  "preset": "holiday-145",
  "presetName": "DONT PANIC",
  "cssVars": { "--color-primary": "#000080", "--color-bg": "#00001a", "..." : "..." },
  "customCss": "\n  .text-logo { font-weight: 900; ... }\n",
  "extensions": [ { "type": "floating", "char": "🧻", "top": "15%", "..." : "..." } ],
  "logoText": "DONT PANIC",
  "logoColors": ["#4285F4", "#34A853", "#FBBC05", "#EA4335"],
  "available": 151,
  "directory": [ { "preset": "yozakura-reverie", "name": "🌸 Yozakura", "..." : "..." } ],
  "dailyIsCommunity": false,
  "apiVersion": "v1",
  "_meta": {
    "sanitized": true,
    "schemaValid": true,
    "timestamp": "2026-05-25T08:24:52.742Z"
  }
}
```

### 回退响应（双平台不可达或校验失败时）

```json
{
  "date": "2026-05-24",
  "presetName": "BUDDHA BIRTHDAY",
  "...": "...",
  "_meta": {
    "sanitized": true,
    "fallback": true,
    "reason": "Current theme failed validation — serving last known safe. Security: pass, Schema: FAIL",
    "flaggedReasons": ["..."],
    "schemaErrors": ["..."],
    "timestamp": "2026-05-25T08:24:52.742Z"
  }
}
```

### 安全机制

| 环节 | 行为 |
|------|------|
| IP 黑名单 | `x-forwarded-for` / `x-real-ip` 命中黑名单 → 403 |
| 违规累计 | 安全扫描不通过时调用 `recordBreach(ip)`，累计违规后自动加入黑名单 |
| 安全事件日志 | 每条违规原因调用 `logSecurityIncident` 写入 KV |
| KV 缓存 | 通过全部检查的版本写入 `theme:last_safe` 作为回退基线 |

## 9. SVG 状态徽章 — `/api/v1/badges/[type]`

**方法：** `GET`
**CORS：** `Access-Control-Allow-Origin: *`
**缓存：** `max-age=60, s-maxage=120, stale-while-revalidate=300`
**Content-Type：** `image/svg+xml`

### 支持的 type

| type | 左侧标签 | 右侧值示例 | 颜色逻辑 |
|------|----------|------------|----------|
| `vercel` | Vercel | `Online (486ms)` | online=绿 / outage=红 / no_data=灰 |
| `netlify` | Netlify | `Online (2075ms)` | 同上 |
| `theme` | Theme | `DONT PANIC` | safe=绿 / unsafe=红 / no_data=灰 |
| `database` | Database | `Healthy` | 无未解决 DB_DOWN 告警=绿 / 有=红 |
| `uptime` | Uptime | `89.7%` | >99%=绿 / >95%=黄 / ≤95%=红 |

### 查询参数

`?debug=1` — 不返回 SVG，返回 JSON 调试信息：

```json
{
  "type": "vercel",
  "logsCount": 78,
  "recentLogsCount": 4,
  "latestVercel": {
    "id": "617af6dc-...",

    "endpoint": "https://themedist.vercel.app/api/v1/today.json",
```

## 10. 网络连通性诊断 — `/api/v1/diagnose`

**方法：** `GET`（含限流）

### 响应

```json
{
  "env": {
    "HTTPS_PROXY": "set",
    "HTTP_PROXY": "set"
  },
  "probes": [
    {
      "url": "themedist.vercel.app/api/v1/today.json",
      "ok": false,
      "ms": 8015,
      "error": "This operation was aborted"
    },
    {
      "url": "www.baidu.com",
      "ok": true,
      "status": 200,
      "ms": 13
    }
  ],
  "summary": "Some unreachable",
  "timestamp": "2026-05-25T08:24:46.267Z"
}
```

| 字段 | 说明 |
|------|------|
| `env.HTTPS_PROXY` | `"set"` / `"not set"` |
| `env.HTTP_PROXY` | `"set"` / `"not set"` |
| `probes[].url` | 已去除协议前缀的域名+路径 |
| `probes[].ok` | boolean |
| `probes[].status` | HTTP 状态码，仅 ok=true 时存在 |
| `probes[].error` | `Error.message` 截断至 120 字符，仅 ok=false 时存在 |
| `summary` | `"All reachable"` / `"Some unreachable"` |

**HTTP 状态码：** 全部可达 → `200`，部分不可达 → `502`。

每个探测 8 秒超时。

## 通用约定

### CORS

以下端点开放跨域（`Access-Control-Allow-Origin: *`）：

- `/api/v1/status` — 含 OPTIONS 预检
- `/api/v1/today-safe`
- `/api/v1/security-status`
- `/api/v1/badges/*`

### 缓存策略

| 端点 | Cache-Control |
|------|---------------|
| `/api/v1/status` | `public, s-maxage=30, stale-while-revalidate=120` |
| `/api/v1/data` | `public, s-maxage=30, stale-while-revalidate=60` |
| `/api/v1/badges/*` | `public, max-age=60, s-maxage=120, stale-while-revalidate=300` |

### 限流

| 端点 | 方法 | 限制 |
|------|------|------|
| `/api/v1/monitor` | GET, POST | 频率限制（`checkRateLimit('monitor')`） |
| `/api/v1/diagnose` | GET | 频率限制（`checkRateLimit('diagnose')`） |

### 鉴权

| 端点 | 方法 | 鉴权方式 |
|------|------|----------|
| `/api/v1/monitor` | POST, DELETE | `Authorization: Bearer <CRON_SECRET>` |

若 `CRON_SECRET` 环境变量未设置，POST/DELETE 回退为允许所有请求（向后兼容）。

### 存储后端

- **KV（Upstash Redis）：** 性能日志、主题快照、告警、状态哈希、安全事件、RUM 计数器、探针结果、锁、IP 黑名单、安全主题缓存
- **文件系统（仅旧数据兼容）：** `/tmp/data`（Vercel）或 `data/`（本地）

### Vercel Cron

| 调度 | 路径 | 说明 |
|------|------|------|
| `0 0 * * *`（每日 00:00 UTC） | `/api/v1/monitor` | 全量监控检查 |
| `0 6 * * *`（每日 06:00 UTC） | `/api/v1/probe` | 多区域拨测 |
